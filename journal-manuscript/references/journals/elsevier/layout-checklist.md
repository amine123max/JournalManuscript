# Elsevier Family Layout Checklist

Use this checklist together with `verification.yaml` before calling the family-level or journal-level layout submission-ready.

## Verification Boundary

- Official template package verified through local Elsevier package assets.
- Journal-specific article type, citation mode, and supplementary-material rules still require confirmation.

## Required Checks

- [ ] `class-file-and-template-variant` Class file and template variant: Confirm whether the target journal should use the elsarticle or CAS branch and which official sample is the closest baseline.
- [ ] `frontmatter-and-keywords` Frontmatter and keywords: Confirm title page, affiliations, abstract, keywords, and any journal-specific metadata required by the target Guide for Authors.
- [ ] `reference-style-alignment` Reference style alignment: Confirm whether the target journal expects numeric or author-year citations and keep the selected Elsevier sample consistent with that guidance.
- [ ] `highlights-and-declarations` Highlights and declarations: Confirm whether highlights, graphical abstract, declaration sections, data statements, or biographies are required, optional, or disallowed.
- [ ] `figure-table-and-submission-assets` Figure, table, and submission assets: Confirm artwork format, table handling, supplementary files, and upload constraints for source submission.
