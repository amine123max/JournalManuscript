# PLOS Family Layout Checklist

Use this checklist together with `verification.yaml` before calling the family-level or journal-level layout submission-ready.

## Verification Boundary

- Official manuscript template package verified through local PLOS assets.
- Title-specific reporting sections and submission rules still require confirmation.

## Required Checks

- [ ] `manuscript-template-and-title-scope` Manuscript template and title scope: Confirm the target PLOS title accepts the manuscript-template path represented by the local package.
- [ ] `frontmatter-and-reporting-sections` Front matter and reporting sections: Confirm abstract, author summary, title-page details, and any title-specific reporting sections follow the target PLOS guidance.
- [ ] `section-flow-and-data-policy` Section flow and data policy: Confirm section hierarchy, data availability, and any ethics or reporting statements match the target PLOS journal.
- [ ] `figure-table-and-submission-assets` Figure, table, and submission assets: Confirm whether figures should remain separate submission assets and whether any specific table or caption rules apply.
- [ ] `references-and-bibliography-style` References and bibliography style: Confirm the target PLOS title's reference formatting and bibliography workflow.
