# Frontiers Family Layout Checklist

Use this checklist together with `verification.yaml` before calling the family-level or journal-level layout submission-ready.

## Verification Boundary

- Official template package verified through local Frontiers package assets.
- Specialty labels, declarations, and title-specific reporting requirements still require confirmation.

## Required Checks

- [ ] `class-file-and-reference-mode` Class file and reference mode: Confirm the target Frontiers title uses the family class path and the appropriate bibliography mode.
- [ ] `frontmatter-and-specialty-metadata` Front matter and specialty metadata: Confirm title, authors, affiliations, correspondence, specialty labels, abstract, and keywords follow the target Frontiers title.
- [ ] `section-flow-and-open-science-rules` Section flow and open science rules: Confirm section hierarchy and all required open-science reporting sections such as data availability, author contributions, conflict of interest, and funding.
- [ ] `figure-table-and-manuscript-rules` Figure, table, and manuscript rules: Confirm figure handling, caption treatment, and manuscript-style layout expectations follow the target Frontiers title.
- [ ] `journal-specific-note` Journal-specific note: Use this family baseline only when the exact Frontiers target journal has not yet been fixed; once the venue is known, tighten the manuscript against that title's live guidance.
