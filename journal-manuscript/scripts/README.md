# Preview Asset Scripts

This directory contains the maintenance scripts that materialize preview assets
for the journal profile library.

## Scripts

- `export_selective_skill_bundle.py`: exports a slim skill bundle that keeps only the shared core files, the selected family profiles, all journals under those families, and the referenced guide or template assets.
- `scaffold_family_manuscript.py`: creates a `paper/` starter workspace from a supported family template baseline, including `main.tex`, `references.bib`, `CAPTION_BANK.md`, and manuscript subfolders.
- `render_profile_preview_assets.py`: renders standardized
  `official_preview.tex/.pdf/.png` assets for folders that are not yet `status: verified`.
- `render_official_template_preview_assets.py`: renders
  `official_preview.tex/.pdf/.png` for verified or blocked targets and refreshes the corresponding
  `verification.yaml` metadata.

## Naming Rule

- Use `render_` for scripts that compile or rasterize presentation assets.
- Use asset-oriented names that describe the output scope, not an incidental
  file format.
- Use `official_preview` as the canonical rendered asset name across the library.
- Reserve verification claims for folders whose `verification.yaml`
  explicitly marks them as `verified`.

## Typical Commands

```powershell
python scripts/render_profile_preview_assets.py
python scripts/render_profile_preview_assets.py --include ieee ieee-access plos-one
python scripts/export_selective_skill_bundle.py --family ieee --archive
python scripts/export_selective_skill_bundle.py --family elsevier --archive
python scripts/scaffold_family_manuscript.py --family ieee --output-dir C:\work\my-paper
python scripts/scaffold_family_manuscript.py --family frontiers --output-dir C:\work\my-frontiers-paper
python scripts/render_official_template_preview_assets.py
python scripts/render_official_template_preview_assets.py --include ieee-access plos-one
```

## Output Contract

- Every family and journal folder should have `profile.md` and `official_preview.*`.
- Journals with a verification record should also have `verification.yaml`.
- Journals whose status is `verified` use `official_preview.*` as official rendered assets, and blocked journals still keep `official_preview.*` as labeled placeholders.
- Every verification target should have a downloaded official guide-source file under `assets/official-templates/guides/<slug>/`.
- Slim bundles generated by `export_selective_skill_bundle.py` should contain the inner `journal-manuscript/` skill folder, the selected family profile directories, all journals that belong to those families, and only the referenced guide or template assets needed by those profiles.
- Family scaffolds generated by `scaffold_family_manuscript.py` should create a usable `paper/` workspace with `main.tex`, `references.bib`, `CAPTION_BANK.md`, `README_PAPER.md`, `figures/`, `tables/`, and `tables/generated/`.
